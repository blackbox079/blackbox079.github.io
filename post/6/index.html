Hugo 是基于 Go 开发的博客框架，文章编译速度快，社区活跃。目前中文相关较少，主题不太多。

安装

macOS/Linux

1

brew install hugo

Windows

hugo 常规版本， hugo-extended Sass/SCSS 扩展版本。这里安装的是 hugo-extended 。

方式一：Chocolatey

1
2
3
4
5
6

# 安装
choco install hugo-extended -confirm
# 更新
choco upgrade hugo-extended
# 卸载
choco uninstall hugo-extended

方式二：Scoop

1

scoop install hugo-extended # 安装

创建站点

1

hugo new site blog

配置

配置文件是根目录下的 config.toml ，一部分是通用的，具体可根据主题的文档来进行配置。这里列出部分通用配置：

baseURL = "https://blog.whatworld.tk" # 域名
languageCode = "zh-cn" # 网站语言

使用

默认模板

修改 archetypes 下的 default.md

1
2
3
4
5
6
7
8
9

---
title: "{{ replace .Name "-" " " | title }}"
date: {{ .Date }}
categories: [""]
tags: [""]
draft: true
---

<!--more-->

其中 <!--more--> 是手动拆分摘要的分隔符。默认情况下，Hugo 会自动将内容的前 70 个词作为摘要，也可以在配置文件 config.toml 中设置 summaryLength 来定制摘要长度。不过显然，使用 <!--more--> 会更为灵活，摘要在它之前，正文在它之后。

常用命令

使用 even 主题，内容应该在 post 文件夹而不是 posts 。

1
2
3

hugo new post/1.md # 新建文件名为1的md文件，命名规则随个人喜好就行
hugo server -D # 启动本地服务器，-D表示显示草稿
hugo # 编译到public

图床

使用 PicGo 上传和管理， imgur 作为博客主图床。

首先注册 imgur，然后 获取 Client ID 。Authorization type 中选择第二项 OAuth 2 authorization without a callback URL ，其余随意填写，通过人机验证提交后可获得 id 和密钥，将 id 复制并填入 PicGo 的 Imgur 图床设置中。因为已经被墙，所以还需设置代理，否则无法上传。本地代理通常是 http://127.0.0.1:端口号 ，具体使用的端口可查看相关软件设置，通常是 1080 或 7890 。

不过目前 PicGo 还没能设置 cdn，可以手动给图片加上 i.staticdn.net/ 。示例：原图地址 https://i.imgur.com/3QrPToe.jpg ，修改后地址 https://i.staticdn.net/i.imgur.com/3QrPToe.jpg 。

上线

GitHub

首先需要有 GitHub 账号，注册GitHub，然后新建仓库，仓库名称为 用户名.github.io， 复制仓库地址后回到终端

1
2
3
4
5
6
7

hugo # 生成可发布文件，默认在public文件夹
cd public
git init # 初始化git本地仓库
git remote add origin 你的GitHub仓库地址 # 添加远程仓库地址
git add -A # 暂存文件
git commit -m "初始提交" # 填写备注信息并提交
git push origin master # 推送到远程服务器

域名

可以在 Freenom 薅一个，开全局代理并确定 ip 所在的国家 搜索并选择你钟意的域名，点击完成 选择12个月免费并继续 输入邮箱并点击验证 回邮箱点击验证链接，注册信息不一定真实，但地址要与ip对应。然后点击View Cart查看购物车，Complete Order完成订单。

失败示例

成功示例

成功后，点击导航栏 Services-My Domains 即可查看管理该域名。

点击 Manage Domain 进行域名管理，dns和名称服务器都由 Cloudflare 处理。我们进行下一步。

CDN

注册 Cloudflare ，添加站点 然后添加 DNS 记录

回到 Freenom ，使用自定义名称服务器，并填入 Cloudflare 提供的服务器地址。

CNAME

在 public 文件夹里增加 CNAME 文件，填入不带协议的域名，示例 blog.whatworld.tk 。将文件提交到 GitHub 仓库，过一会即可生效。
